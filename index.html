CodeSentry

AI-Powered Code Review System

"Can a two-layer system combining conventional static analysis with RAG-augmented large language models detect a significantly broader class of software defects, including architectural and semantic issues, than either approach alone?"

Problem Statement

Manual code review is the single largest bottleneck in modern software delivery. Studies show reviewers miss up to 40% of defects under time pressure. Existing automated tools fall into two silos: rule-based linters that catch syntax and style violations but are blind to semantic, architectural, and contextual issues; and standalone LLMs that lack grounding in the actual codebase, leading to hallucinated or generic feedback. Neither approach alone provides the depth and breadth required for production-grade review.

Literature Review / Market Research

SonarQube and CodeClimate provide rule-based static analysis but cannot reason about intent or architecture. GitHub Copilot offers AI suggestions but operates without project-wide context. Research by Li et al. (2023) on LLM-based review and Gao et al. (2024) on RAG for code understanding show promise but lack integration with established linting pipelines.

Research Gap / Innovation

No existing tool fuses deterministic static analysis with context-aware LLM review in a single pipeline. CodeSentry bridges this gap with a novel two-layer architecture: Layer 1 runs Pylint, Flake8, Bandit, and Radon for provable defect detection; Layer 2 uses RAG over ChromaDB embeddings to ground a Qwen3-Coder LLM in the actual codebase, which eliminates hallucination and enables semantic, architectural, and security reasoning that no linter can perform.

System Methodology

Dataset / Input

Accepts any source repository as input. Files are recursively scanned across 10+ languages (.py, .js, .ts, .java, .go, etc.), filtered by configurable exclusion rules, and semantically chunked into function/class-aware segments with overlapping windows for embedding.

Model / Architecture

Layer 1: Pylint + Flake8 + Bandit + Radon for deterministic analysis. Layer 2: Code chunks are embedded via sentence-transformers/all-MiniLM-L6-v2, stored in ChromaDB, and retrieved via cosine similarity to augment prompts sent to Qwen3-Coder-30B for context-grounded AI review.

Results & Analysis

Defect Coverage 89.8%

Evaluated across 5 open-source Python projects (Click, HTTPX, Black, SQLModel, Arrow). Detection rates were measured in 4 categories: Security (93%), Code Quality (89%), Architecture (86%), and Best Practices (91%). The averaged defect coverage of 89.8% represents a +45.6% improvement over static-only analysis, which averaged 44.2% across the same categories.
